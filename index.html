1.1. Co to jest SQL?

SQL (z ang. Structured Query Language) to strukturalny język zapytań. Jest to światowy standard używany do komunikacji z relacyjnymi bazami danych.

Dlaczego warto znać SQL?

  • Pozwala na szybkie wyszukiwanie informacji w ogromnych zbiorach danych.
  • Jest uniwersalny – nauczysz się go raz i wykorzystasz w MySQL, PostgreSQL, MS Access czy SQL Server.
  • Pozwala nie tylko czytać dane, ale też je dodawać, zmieniać i usuwać.
Ważne: SQL jest językiem deklaratywnym. Oznacza to, że mówisz bazie CO chcesz osiągnąć (np. "daj mi nazwiska uczniów z klasy 2B"), a nie JAK komputer ma to technicznie zrobić.

Podstawowe operacje (CRUD)

Większość pracy w SQL sprowadza się do czterech operacji:

Operacja Polecenie SQL Opis
Create INSERT Dodawanie nowych danych
Read SELECT Odczytywanie danych (najczęstsze)
Update UPDATE Zmiana istniejących danych
Delete DELETE Usuwanie danych

Przykłady

Oto jak wygląda przykładowe zapytanie, które pobiera imiona i nazwiska wszystkich pracowników:

SELECT Imie, Nazwisko FROM Pracownicy;

1.3. Pierwszy SELECT i Aliasy

W poprzedniej lekcji używaliśmy SELECT *, aby zobaczyć wszystko. W praktyce jednak rzadko potrzebujemy wszystkich kolumn. Zazwyczaj wybieramy tylko te dane, które są nam w danej chwili potrzebne.

Wybieranie konkretnych kolumn

Zamiast gwiazdki, po słowie SELECT wymieniamy nazwy kolumn po przecinku.

-- Pobranie tylko imion i nazwisk uczniów
SELECT imie, nazwisko FROM Uczniowie;

Aliasy (Zmienianie nazw kolumn w wyniku)

Często nazwy kolumn w bazie są krótkie lub techniczne (np. id_produktu). Możemy użyć słowa kluczowego AS, aby w tabeli wynikowej wyświetliła się nasza własna nazwa.

-- Użycie aliasów dla lepszej czytelności
SELECT nazwa AS Produkt, cena AS [Cena detaliczna] 
FROM Produkty;
Wskazówka: Jeśli Twoja nowa nazwa (alias) zawiera spację, musisz ją ująć w nawiasy kwadratowe [ ] lub cudzysłów (w zależności od systemu bazy danych).

Przykłady rozwiązane

Przykład: Wyświetl listę uczniów, ale kolumny nazwij "Osoba" (imię i nazwisko razem - w SQL robimy to przez przecinek w SELECT lub łączenie tekstów) oraz "Klasa".

SELECT imie, nazwisko, klasa AS [Grupa szkolna] FROM Uczniowie;

2.1. Filtrowanie danych - Klauzula WHERE

Klauzula WHERE pozwala ograniczyć liczbę wyświetlanych wierszy do tych, które spełniają określone przez nas warunki. To najważniejsze narzędzie do przeszukiwania bazy.

Podstawowe operatory porównania

W SQL używamy standardowych znaków matematycznych:

  • = (Równe)
  • <> lub != (Różne od)
  • < , > (Mniejsze, Większe)
  • <= , >= (Mniejsze/Większe lub równe)

Przykłady użycia

1. Filtrowanie liczb

Chcemy zobaczyć tylko drogie produkty (cena powyżej 1000 zł):

SELECT * FROM Produkty WHERE cena > 1000;

2. Filtrowanie tekstu

Ważne: Teksty w SQL zawsze piszemy w pojedynczym cudzysłowie (apostrofie) ' '.

-- Wyświetl tylko uczniów z klasy 2B
SELECT imie, nazwisko FROM Uczniowie WHERE klasa = '2B';
Uważaj na wielkość liter! W wielu bazach danych (jak SQLite w kompilatorze online) '2B' to nie to samo co '2b'. Pisz warunki dokładnie tak, jak dane zostały wprowadzone do tabeli.

Zadania do samodzielnego wykonania

Zadanie 1: Wyświetl wszystkie dane o uczniach, którzy są mężczyznami (kolumna plec ma wartość 'M').

Kliknij, aby zobaczyć odpowiedź
SELECT * FROM Uczniowie WHERE plec = 'M';

Zadanie 2: Wyświetl nazwy produktów, których mamy w magazynie (ilosc_magazyn) dokładnie 5 sztuk.

Kliknij, aby zobaczyć odpowiedź
SELECT nazwa FROM Produkty WHERE ilosc_magazyn = 5;

Zadanie 3: Wyświetl wszystkie oceny z tabeli Oceny, które są niższe niż 4.

Kliknij, aby zobaczyć odpowiedź
SELECT * FROM Oceny WHERE ocena < 4;

2.2. Operatory logiczne: AND, OR, NOT

W rzeczywistości rzadko szukamy danych spełniających tylko jeden warunek. Zazwyczaj potrzebujemy kombinacji, np. "produkty z kategorii Elektronika, które kosztują mniej niż 200 zł". Do tego służą operatory logiczne.

1. Operator AND (I)

Zwraca rekordy, które spełniają wszystkie podane warunki jednocześnie.

-- Szukamy uczniów z klasy 2B, którzy są kobietami
SELECT * FROM Uczniowie 
WHERE klasa = '2B' AND plec = 'K';

2. Operator OR (LUB)

Zwraca rekordy, które spełniają przynajmniej jeden z podanych warunków.

-- Szukamy uczniów z klasy 2B lub z klasy 3A
SELECT * FROM Uczniowie 
WHERE klasa = '2B' OR klasa = '3A';

3. Operator NOT (NIE)

Zwraca rekordy, które nie spełniają podanego warunku (zaprzeczenie).

-- Szukamy wszystkich produktów, które NIE należą do kategorii AGD
SELECT * FROM Produkty 
WHERE NOT kategoria = 'AGD';
Uwaga na nawiasy! Jeśli łączysz AND oraz OR w jednym zapytaniu, używaj nawiasów, aby określić kolejność sprawdzania warunków (podobnie jak w matematyce).

Przykłady rozwiązane

Przykład: Wyświetl produkty z kategorii 'Elektronika', których cena jest wyższa niż 100 zł, ale mniejsza niż 2000 zł.

SELECT * FROM Produkty 
WHERE kategoria = 'Elektronika' AND cena > 100 AND cena < 2000;

Zadania do samodzielnego wykonania

Zadanie 1: Wyświetl uczniów, którzy nazywają się 'Nowak' i są z klasy '2B'.

Kliknij, aby zobaczyć odpowiedź
SELECT * FROM Uczniowie WHERE nazwisko = 'Nowak' AND klasa = '2B';

Zadanie 2: Wyświetl oceny (tabela Oceny), które są równe 5 LUB 6.

Kliknij, aby zobaczyć odpowiedź
SELECT * FROM Oceny WHERE ocena = 5 OR ocena = 6;

Zadanie 3: Wyświetl produkty, które są z kategorii 'Elektronika' i mają cenę różną od 150.00 zł.

Kliknij, aby zobaczyć odpowiedź
SELECT * FROM Produkty WHERE kategoria = 'Elektronika' AND cena <> 150.00;

2.3. Wyszukiwanie wzorców - Operator LIKE

Operator LIKE jest używany, gdy nie szukamy konkretnej wartości, ale tekstu pasującego do pewnego wzorca (np. nazwisk na literę 'P' lub telefonów zaczynających się od '605').

Standard Accessa (Twój priorytet):
W Microsoft Access używamy dwóch głównych znaków wieloznacznych:
  • * (gwiazdka) – zastępuje dowolną liczbę znaków.
  • ? (pytajnik) – zastępuje dokładnie jeden znak.
⚠️ Uwaga techniczna (Kompilator Online):
Jeśli ćwiczysz teraz w przeglądarce (NextLeap), pamiętaj, że tam zamiast gwiazdki * musisz wpisać procent %, a zamiast pytajnika ? – podkreślnik _.

Przykłady w składni Access

1. Gwiazdka (*) – wiele znaków

Szukamy wszystkich klientów, których numer telefonu zaczyna się od '605':

-- Składnia Access
SELECT * FROM Klienci WHERE telefon LIKE '605*';

-- Jeśli testujesz ONLINE:
-- SELECT * FROM Klienci WHERE telefon LIKE '605%';

2. Pytajnik (?) – jeden znak

Szukamy nazwisk, które mają dowolną literę na drugim miejscu, np. 'Kowalski', 'Kawalec' (wzorzec K?w*):

-- Składnia Access
SELECT * FROM Uczniowie WHERE nazwisko LIKE 'K?w*';

Zadania do samodzielnego wykonania (składnia Access)

Zadanie 1: Wyświetl listę piłkarzy (wszystkie dane), których numer klubowy kończy się na cyfrę 7.

Kliknij, aby zobaczyć odpowiedź (Access)
SELECT * FROM Piłkarze WHERE numer_klubowy LIKE '*7';

Zadanie 2: Znajdź wszystkie zamówienia z marca. Data jest zapisana w formacie RRRR-MM-DD (użyj wzorca z gwiazdkami).

Kliknij, aby zobaczyć odpowiedź (Access)
SELECT * FROM Zamowienia WHERE data LIKE '*-03-*';

Zadanie 3: Wyświetl uczniów, których nazwisko zaczyna się na literę 'P'.

Kliknij, aby zobaczyć odpowiedź (Access)
SELECT * FROM Uczniowie WHERE nazwisko LIKE 'P*';

2.4. Praca na zbiorach i zakresach - IN oraz BETWEEN

Zamiast pisać długie łańcuchy warunków typu id=1 OR id=2 OR id=3, możemy użyć operatorów zbiorowych. Dzięki temu zapytania są krótsze i czytelniejsze.

1. Operator IN (W zbiorze)

Pozwala sprawdzić, czy wartość kolumny znajduje się na liście wymienionej w nawiasach.

-- Wyświetl uczniów z klas 1A, 2B i 3A
SELECT * FROM Uczniowie 
WHERE klasa IN ('1A', '2B', '3A');

2. Operator BETWEEN (Pomiędzy)

Służy do filtrowania danych w domkniętym przedziale (od - do). Najczęściej używany przy liczbach i datach.

-- Wyświetl produkty w cenie od 100 do 1000 zł
SELECT * FROM Produkty 
WHERE cena BETWEEN 100 AND 1000;
Wskazówka Access: Operator BETWEEN obejmuje wartości graniczne. Jeśli szukasz daty w Accessie, pamiętaj, że daty często zapisujemy w kratkach, np. BETWEEN #2023-01-01# AND #2023-03-31#.

Przykłady z Twoich materiałów

Przykład: Wyświetl oceny, które są równe 2, 3 lub 4.

SELECT * FROM Oceny WHERE ocena IN (2, 3, 4);

Zadania do samodzielnego wykonania

Zadanie 1: Wyświetl wszystkie produkty, których id_produktu to 1, 3 lub 5.

Kliknij, aby zobaczyć odpowiedź
SELECT * FROM Produkty WHERE id_produktu IN (1, 3, 5);

Zadanie 2: Znajdź oceny wystawione pomiędzy 1 a 20 października 2023 roku.

Kliknij, aby zobaczyć odpowiedź
SELECT * FROM Oceny WHERE data_wystawienia BETWEEN '2023-10-01' AND '2023-10-20';

Zadanie 3: Wyświetl produkty, które NIE mają ceny w zakresie od 100 do 500 zł (użyj NOT BETWEEN).

Kliknij, aby zobaczyć odpowiedź
SELECT * FROM Produkty WHERE cena NOT BETWEEN 100 AND 500;

2.5. Funkcje dat - YEAR, MONTH, DAY

W bazach danych MS Access często nie potrzebujemy pełnej daty (np. 2023-10-15), a jedynie konkretną informację: rok, miesiąc lub dzień. Służą do tego wbudowane funkcje, które "wycinają" interesujący nas fragment z pola typu data.

Kluczowe funkcje i ich działanie

  • YEAR(pole_daty) – zwraca rok jako liczbę (np. 2023).
  • MONTH(pole_daty) – zwraca numer miesiąca jako liczbę (od 1 do 12).
  • DAY(pole_daty) – zwraca dzień miesiąca jako liczbę (od 1 do 31).

Przykłady praktyczne

1. Filtrowanie po roku

Wyświetl wszystkich uczniów urodzonych w 2008 roku:

SELECT * FROM Uczniowie 
WHERE YEAR(data_urodzenia) = 2008;

2. Filtrowanie po miesiącu i dniu

Wyświetl oceny wystawione w maju (dowolnego roku):

SELECT * FROM Oceny 
WHERE MONTH(data_wystawienia) = 5;
Łączenie warunków: Możesz precyzyjnie określić termin, łącząc funkcje za pomocą AND. Zapytanie: WHERE YEAR(data) = 2023 AND MONTH(data) = 12 znajdzie rekordy tylko z grudnia 2023 roku.

Zadania do samodzielnego wykonania

Zadanie 1: Wyświetl listę ocen (wszystkie kolumny), które zostały wystawione w 2024 roku.

Kliknij, aby zobaczyć odpowiedź
SELECT * FROM Oceny WHERE YEAR(data_wystawienia) = 2024;

Zadanie 2: Znajdź uczniów, którzy obchodzą urodziny w pierwszym dniu dowolnego miesiąca.

Kliknij, aby zobaczyć odpowiedź
SELECT * FROM Uczniowie WHERE DAY(data_urodzenia) = 1;

2.6. Sortowanie i wybieranie rekordów (TOP)

Samo wyfiltrowanie danych to połowa sukcesu. Często musimy je uporządkować lub wyciągnąć tylko te rekordy, które znajdują się na samym szczycie listy (np. najdroższy produkt).

Klauzula ORDER BY:

Służy do sortowania wyników według jednej lub kilku kolumn.

  • ASC – sortowanie rosnąco (domyślne, od A do Z, od 1 do 100).
  • DESC – sortowanie malejąco (od Z do A, od 100 do 1).

1. Sortowanie podstawowe

Wyświetlmy uczniów posortowanych alfabetycznie według nazwiska:

SELECT * FROM Uczniowie ORDER BY Nazwisko ASC;

2. Wybieranie rekordów z góry (TOP)

W MS Access słowo TOP pozwala ograniczyć liczbę wyświetlanych wierszy do tych, które po przesortowaniu znalazły się na początku.

Przykład: Najwyższa ocena

Aby znaleźć ucznia z najwyższą oceną, musimy najpierw posortować oceny malejąco, a potem wziąć tylko pierwszy rekord:

SELECT TOP 1 Nazwisko, Ocena 
FROM Oceny 
ORDER BY Ocena DESC;

3. Najstarszy lub najmłodszy (Sortowanie dat)

Sortowanie dat bywa mylące, ale pamiętaj:

  • ORDER BY Data_Urodzenia ASC – pokaże najstarsze osoby (najwcześniejsze daty).
  • ORDER BY Data_Urodzenia DESC – pokaże najmłodsze osoby (najpóźniejsze daty).
Zadanie kontrolne:

Napisz zapytanie, które wyświetli tylko jeden najtańszy produkt z tabeli Produkty.

Pokaż rozwiązanie

Używamy TOP 1 oraz sortowania rosnącego (ASC):

SELECT TOP 1 Nazwa_Produktu, Cena FROM Produkty ORDER BY Cena ASC;
Ważne: Klauzula TOP zawsze występuje bezpośrednio po słowie SELECT, natomiast ORDER BY zawsze na samym końcu zapytania.

3.1. Funkcje Agregujące (SUM, AVG, COUNT...)

Do tej pory wybieraliśmy konkretne wiersze z bazy. Funkcje agregujące pozwalają na coś więcej – wykonują obliczenia na wielu rekordach i zwracają pojedynczy wynik (np. sumę wszystkich wydatków lub średnią ocen klasy).

Najważniejsze funkcje w MS Access SQL:
  • COUNT(*) lub COUNT(kolumna) – zlicza liczbę rekordów.
  • SUM(kolumna) – sumuje wartości (tylko dla liczb).
  • AVG(kolumna) – oblicza średnią arytmetyczną (tylko dla liczb).
  • MAX(kolumna) i MIN(kolumna) – znajduje wartość największą i najmniejszą.

1. Zliczanie rekordów (COUNT)

Funkcja COUNT jest najczęściej używana. Możesz policzyć wszystkich uczniów w tabeli:

SELECT COUNT(*) AS [Liczba wszystkich uczniów] FROM Uczniowie;

2. Średnia i Suma (AVG, SUM)

Działają tylko na polach liczbowych (np. cena, ocena, wiek). Przykład obliczenia średniej ceny produktów:

SELECT AVG(cena) AS SredniaCena FROM Produkty;

3. Filtrowanie przed agregacją

Możesz łączyć funkcje agregujące z klauzulą WHERE. Na przykład, aby policzyć tylko uczniów z klasy '2B':

SELECT COUNT(*) FROM Uczniowie WHERE klasa = '2B';
Mini-zadanie:

Jak sprawdzić, jaka jest najwyższa ocena wystawiona w tabeli Oceny?

Pokaż podpowiedź i rozwiązanie

Używamy funkcji MAX na kolumnie ocena:

SELECT MAX(ocena) FROM Oceny;
Ważna uwaga o Aliasach (AS):

W Accessie wynik funkcji agregującej często nazywa się "Expr1001". Używaj AS NowaNazwa, aby wynik był czytelny w raporcie!

3.2. Grupowanie danych (GROUP BY)

Sama funkcja AVG powie Ci, jaka jest średnia ocen w całej szkole. Ale co, jeśli chcesz poznać średnią dla każdej klasy z osobna? Do tego służy klauzula GROUP BY.

Zasada działania:

GROUP BY "rozcina" tabelę na mniejsze paczki według wartości w wybranej kolumnie, a następnie wykonuje obliczenia (np. COUNT lub SUM) dla każdej z tych paczek oddzielnie.

1. Przykład: Liczba osób w klasach

Aby sprawdzić, ilu uczniów jest w każdej z klas, musimy pogrupować ich według kolumny klasa:

SELECT klasa, COUNT(*) AS LiczbaUczniow 
FROM Uczniowie 
GROUP BY klasa;

2. Przykład: Średnia cena w kategoriach

W tabeli Produkty możemy sprawdzić średnią cenę dla każdej kategorii (RTV, AGD, Dom):

SELECT kategoria, AVG(cena) AS SredniaCena 
FROM Produkty 
GROUP BY kategoria;
Złota zasada GROUP BY:

Każda kolumna, która pojawia się w SELECT, a nie jest częścią funkcji agregującej (jak COUNT czy SUM), musi znaleźć się w GROUP BY. W przeciwnym razie Access wyświetli błąd!

3. Grupowanie z sortowaniem

Możesz połączyć grupowanie z sortowaniem, aby zobaczyć np. klasy z największą liczbą uczniów na górze:

SELECT klasa, COUNT(*) AS Liczba 
FROM Uczniowie 
GROUP BY klasa 
ORDER BY COUNT(*) DESC;
Zadanie do przemyślenia:

Masz tabelę Oceny z kolumnami przedmiot oraz ocena. Jak sprawdzić, jaka jest najniższa ocena z każdego przedmiotu?

Sprawdź poprawne zapytanie

Musimy pogrupować dane po przedmiocie i użyć funkcji MIN:

SELECT przedmiot, MIN(ocena) FROM Oceny GROUP BY przedmiot;

Lekcja 3.3: Filtrowanie grup (HAVING)

Do tej pory filtrowaliśmy pojedyncze wiersze za pomocą WHERE. Co jednak zrobić, gdy chcemy przefiltrować wyniki obliczeń? Na przykład wyświetlić tylko te kategorie produktów, w których średnia cena jest wyższa niż 100 zł?

Najważniejsza różnica:
  • WHERE – filtruje dane przed grupowaniem (pojedyncze rekordy).
  • HAVING – filtruje dane po grupowaniu (wyniki funkcji takich jak SUM czy COUNT).

1. Przykład: Tylko liczne klasy

Chcemy wyświetlić tylko te klasy, w których uczy się więcej niż 5 osób:

SELECT klasa, COUNT(*) AS Liczba 
FROM Uczniowie 
GROUP BY klasa 
HAVING COUNT(*) > 5;

2. Przykład: Wysoka średnia ocen

Załóżmy, że interesują nas tylko przedmioty, z których średnia ocen jest wyższa niż 4.5:

SELECT przedmiot, AVG(ocena) AS Srednia 
FROM Oceny 
GROUP BY przedmiot 
HAVING AVG(ocena) > 4.5;
Kolejność w zapytaniu (Bardzo ważne!):

W SQL nie można zamieniać kolejności klauzul. Poprawny schemat to:

  1. SELECT (co wybierasz)
  2. FROM (z jakiej tabeli)
  3. WHERE (warunek dla wierszy)
  4. GROUP BY (grupowanie)
  5. HAVING (warunek dla grup)
  6. ORDER BY (sortowanie końcowe)
Zadanie kontrolne:

Napisz zapytanie, które wyświetli nazwy kategorii produktów, ale tylko takich, których suma wartości w magazynie (SUM) przekracza 10 000 zł.

Kliknij, aby zobaczyć rozwiązanie
SELECT kategoria, SUM(wartosc) FROM Produkty GROUP BY kategoria HAVING SUM(wartosc) > 10000;

4.1. Idea relacji i klucze (PK, FK)

W profesjonalnych bazach danych nie trzymamy wszystkiego w jednej, ogromnej tabeli. Powodowałoby to ogromny bałagan i powtarzanie tych samych informacji (np. danych o wydawcy przy każdej książce).

Dlaczego dzielimy dane?
  • Oszczędność miejsca: Nazwę klasy zapisujemy raz w tabeli Klasy, a nie 30 razy przy każdym uczniu.
  • Łatwa aktualizacja: Jeśli klasa zmieni wychowawcę, poprawiamy to w jednym miejscu.
  • Porządek: Każda tabela odpowiada za konkretny typ informacji (Uczniowie, Oceny, Przedmioty).

1. Klucz Podstawowy (Primary Key - PK)

To unikalny identyfikator każdego rekordu w tabeli. Nie może się powtarzać i nie może być pusty. Najczęściej jest to pole ID.

-- Przykład w tabeli Autorzy
ID_Autora | Imie | Nazwisko
---------------------------
1         | Adam | Mickiewicz  <-- 1 to Klucz Podstawowy

2. Klucz Obcy (Foreign Key - FK)

To "pomost" do innej tabeli. Jest to kolumna w jednej tabeli, która przechowuje wartość klucza podstawowego z innej tabeli.

Przykład praktyczny:

Mamy tabelę Klasy (PK to ID_klasy) oraz tabelę Uczniowie. Aby przypisać ucznia do klasy, w tabeli uczniów tworzymy klucz obcy:

Tabela Klasy:
ID_klasy: 1 | Nazwa: 2B

Tabela Uczniowie:
ID_ucznia: 105 | Nazwisko: Kowalski | ID_klasy: 1 (FK)
Rodzaje relacji:
  • Jeden-do-wielu (1:N): Najczęstsza. Jeden autor może napisać wiele książek, ale książka ma (zazwyczaj) jednego autora.
  • Wiele-do-wielu (M:N): Jeden uczeń ma wielu nauczycieli, a jeden nauczyciel ma wielu uczniów. (Wymaga tabeli pośredniej!).
  • Jeden-do-jeden (1:1): Rzadka. Np. jeden obywatel ma jeden numer PESEL.
Pytanie kontrolne:

Które pole w tabeli Oceny prawdopodobnie jest Kluczem Obcym (FK), który pozwala sprawdzić, kto dostał daną ocenę?

Sprawdź odpowiedź

Jest to pole ID_ucznia (lub nr_legitymacji), które odsyła nas do tabeli Uczniowie.

4.2. Łączenie wewnętrzne (INNER JOIN)

Skoro wiemy już, że dane są dzielone na tabele (np. Uczniowie i Klasy), musimy nauczymy się je "sklejać", aby wyświetlić pełną informację: imię ucznia obok pełnej nazwy jego klasy.

Co robi INNER JOIN?

Szuka wspólnych wartości w obu tabelach (zazwyczaj Klucza Podstawowego w jednej i Klucza Obcego w drugiej). Jeśli znajdzie dopasowanie – łączy wiersze w jeden długi rekord. Jeśli nie znajdzie – pomija dany wiersz.

1. Składnia zapytania

Podstawowy schemat łączenia dwóch tabel wygląda tak:

SELECT Tabela1.kolumna, Tabela2.kolumna
FROM Tabela1
INNER JOIN Tabela2 ON Tabela1.Klucz_PK = Tabela2.Klucz_FK;

2. Przykład: Uczniowie i ich Klasy

Chcemy wyświetlić Nazwisko ucznia oraz Nazwę klasy, do której uczęszcza:

SELECT Uczniowie.Nazwisko, Klasy.Nazwa_Klasy
FROM Uczniowie
INNER JOIN Klasy ON Uczniowie.ID_klasy = Klasy.ID_klasy;

3. Dobra praktyka: Aliasy tabel

Pisanie pełnych nazw tabel (np. Uczniowie.Nazwisko) bywa męczące. Możemy nadać tabelom krótkie ksywki (aliasy):

SELECT u.Nazwisko, k.Nazwa_Klasy
FROM Uczniowie AS u
INNER JOIN Klasy AS k ON u.ID_klasy = k.ID_klasy;
Na co uważać w Accessie?

Jeśli łączysz tabele w widoku SQL MS Access, pamiętaj, że Access bardzo lubi nawiasy przy łączeniach, ale przy prostym INNER JOIN dwóch tabel nie są one wymagane. Problem zacznie się przy trzech tabelach (o czym w następnej lekcji!).

Zadanie do samodzielnego wykonania:

Masz tabelę Ksiazki (z polem ID_autora) oraz tabelę Autorzy (z polem ID_autora). Napisz zapytanie, które wyświetli Tytuł książki oraz Nazwisko jej autora.

Pokaż poprawne zapytanie
SELECT Ksiazki.Tytul, Autorzy.Nazwisko
FROM Ksiazki
INNER JOIN Autorzy ON Ksiazki.ID_autora = Autorzy.ID_autora;

4.3. Łączenie wielu tabel (3+)

Często jedno połączenie to za mało. Aby wyświetlić nazwisko ucznia, nazwę przedmiotu i wystawioną mu ocenę, musimy "skleić" ze sobą aż trzy tabele: Uczniowie, Oceny oraz Przedmioty.

⚠️ Pułapka MS Access (Nawiasy):

W przeciwieństwie do MySQL czy PostgreSQL, MS Access wymaga, aby każde kolejne łączenie (JOIN) było zamknięte w nawiasach. Jeśli łączysz 3 tabele, pierwsze łączenie musi być w nawiasie!

1. Schemat łączenia 3 tabel

Zauważ, gdzie znajdują się nawiasy – obejmują one całe pierwsze połączenie INNER JOIN:

SELECT kolumny
FROM (Tabela1 
INNER JOIN Tabela2 ON Tabela1.PK = Tabela2.FK)
INNER JOIN Tabela3 ON Tabela2.PK = Tabela3.FK;

2. Przykład praktyczny: Uczeń i jego oceny z przedmiotów

Chcemy zobaczyć: Nazwisko, Nazwę Przedmiotu oraz Wartość Oceny. Łączymy tabelę Uczniowie z Oceny, a następnie Oceny z Przedmioty:

SELECT u.Nazwisko, p.Nazwa_Przedmiotu, o.Ocena
FROM (Uczniowie AS u 
INNER JOIN Oceny AS o ON u.ID_ucznia = o.ID_ucznia)
INNER JOIN Przedmioty AS p ON o.ID_przedmiotu = p.ID_przedmiotu;

3. Jak to zrozumieć "na chłopski rozum"?

  1. Najpierw SQL łączy Ucznia z jego Ocenami (mamy listę: Kowalski - 5, Kowalski - 4).
  2. Następnie do każdej tej oceny "dokleja" Nazwę Przedmiotu na podstawie ID_przedmiotu zapisanego przy ocenie.
Wskazówka egzaminacyjna:

Jeśli musisz połączyć 4 tabele, dodajesz kolejny nawias na samym początku i kolejny INNER JOIN na końcu. Zawsze o jeden nawias mniej niż liczba tabel!

Zadanie "Gwiazdka":

Spróbuj napisać zapytanie łączące 3 tabele: Autorzy, Ksiazki oraz Wydawnictwa, aby wyświetlić Nazwisko autora, Tytuł książki i Nazwę wydawnictwa.

Pokaż rozwiązanie
SELECT a.Nazwisko, k.Tytul, w.Nazwa_Wydawcy
FROM (Autorzy AS a
INNER JOIN Ksiazki AS k ON a.ID_autora = k.ID_autora)
INNER JOIN Wydawnictwa AS w ON k.ID_wydawcy = w.ID_wydawcy;

5.1. Operacje na zbiorach (UNION)

Do tej pory używaliśmy JOIN, aby łączyć tabele "w poziomie" (doklejając kolumny). Operator UNION pozwala na łączenie tabel "w pionie" – czyli doklejanie wierszy z jednego zapytania do drugiego.

Kiedy używamy UNION?

Gdy chcemy stworzyć jedną wspólną listę z dwóch różnych tabel, np. listę wszystkich osób w szkole (zarówno uczniów, jak i nauczycieli) w jednej kolumnie.

1. Zasady bezpiecznego łączenia

Aby operator UNION zadziałał, musisz spełnić trzy warunki:

  • Oba zapytania SELECT muszą mieć tę samą liczbę kolumn.
  • Kolumny na tych samych pozycjach muszą mieć podobne typy danych (np. tekst z tekstem, liczba z liczbą).
  • Nazwy kolumn w wyniku końcowym zostaną przejęte z pierwszego zapytania.

2. Przykład: Wspólna lista kontaktowa

Połączmy nazwiska z dwóch różnych tabel w jeden wynik:

SELECT Nazwisko, Telefon FROM Uczniowie
UNION
SELECT Nazwisko, Telefon FROM Nauczyciele;

3. UNION vs UNION ALL

To bardzo ważna różnica, o którą często pytają na egzaminach:

  • UNION – usuwa duplikaty. Jeśli to samo nazwisko występuje w obu tabelach, w wyniku pojawi się tylko raz.
  • UNION ALL – wyświetla wszystko, nawet powtórki. Jest znacznie szybszy, bo baza nie musi tracić czasu na szukanie duplikatów.
Wskazówka:

Jeśli chcesz posortować wynik całego połączenia, klauzulę ORDER BY dopisz tylko raz, na samym końcu (po ostatnim SELECT).

Zadanie "Wielka Lista":

Masz tabelę Klienci_Polska oraz Klienci_Zagranica. Napisz zapytanie, które wyświetli jedną listę wszystkich miast, w których masz klientów (bez duplikatów).

Pokaż rozwiązanie

Używamy zwykłego UNION, aby pozbyć się powtarzających się nazw miast:

SELECT Miasto FROM Klienci_Polska
UNION
SELECT Miasto FROM Klienci_Zagranica;

5.2. Podzapytania (Subqueries)

Podzapytanie to instrukcja SELECT umieszczona wewnątrz innego zapytania. Wynik "wewnętrznego" zapytania jest przekazywany do "zewnętrznego" jako gotowa wartość do porównania.

Kiedy używamy podzapytań?

Gdy warunek filtrowania zależy od obliczeń, których nie znamy z góry. Podzapytanie zawsze musi być zamknięte w nawiasach okrągłych.

1. Podzapytanie zwracające jedną wartość

Chcemy wyświetlić produkty, których cena jest wyższa niż średnia cena wszystkich produktów w sklepie:

SELECT Nazwa_Produktu, Cena
FROM Produkty
WHERE Cena > (SELECT AVG(Cena) FROM Produkty);

2. Podzapytanie z operatorem IN

Możemy też użyć podzapytania, które zwraca listę wartości. Na przykład: wyświetl uczniów, którzy dostali przynajmniej jedną ocenę '6':

SELECT Nazwisko, Imie
FROM Uczniowie
WHERE ID_ucznia IN (SELECT ID_ucznia FROM Oceny WHERE Ocena = 6);

3. Gdzie jeszcze można wstawić podzapytanie?

  • W klauzuli SELECT: Aby wyświetlić dodatkową kolumnę z obliczeniem obok każdego wiersza.
  • W klauzuli FROM: Traktując wynik podzapytania jako tymczasową tabelę.
Ważna uwaga o wydajności:

Podzapytania są potężne, ale mogą spowalniać bazę, jeśli są bardzo złożone. Często ten sam wynik można osiągnąć za pomocą INNER JOIN, który zazwyczaj działa szybciej.

Zadanie "Ponadprzeciętni":

Napisz zapytanie, które wyświetli nazwiska uczniów, których wiek jest mniejszy niż średni wiek wszystkich osób w tabeli Uczniowie.

Pokaż rozwiązanie
SELECT Nazwisko FROM Uczniowie
WHERE Wiek < (SELECT AVG(Wiek) FROM Uczniowie);

5.3. Zaawansowane narzędzia Access – Parametry i Crosstab

MS Access to nie tylko silnik bazy danych, ale całe środowisko raportowe. W tej lekcji nauczysz się, jak sprawić, by Twoje zapytania były interaktywne oraz jak tworzyć tabele przestawne bezpośrednio w SQL.

---

1. Zapytania Parametryczne – Interakcja z użytkownikiem

Zamiast tworzyć 10 oddzielnych zapytań dla każdej klasy (2A, 2B, 3A...), tworzysz jedno, które przy każdym uruchomieniu prosi o podanie wartości.

Mechanika działania:

Access szuka w kodzie SQL nazw w nawiasach kwadratowych [...]. Jeśli tekst wewnątrz nie pasuje do żadnej nazwy kolumny ani tabeli, program interpretuje to jako komunikat dla użytkownika i wyświetla okienko dialogowe.

Przykład: Filtrowanie zakresu dat

Możesz poprosić o dwie wartości, aby wyświetlić sprzedaż z konkretnego okresu:

SELECT ID_Zamowienia, Data_Sprzedazy, Kwota
FROM Zamowienia
WHERE Data_Sprzedazy BETWEEN [Podaj datę początkową (RRRR-MM-DD):] 
                         AND [Podaj datę końcową:];
Wskazówka: Pamiętaj, aby tekst w nawiasie był precyzyjny (np. dodaj informację o formacie daty). Dzięki temu osoba korzystająca z bazy nie popełni błędu.
---

2. Zapytania Krzyżowe (CROSSTAB) – Raporty macierzowe

To "święty graal" raportowania w Accessie. Zapytanie krzyżowe (Crosstab) przekształca wiersze w kolumny. Jest to idealne, gdy chcesz podsumować np. oceny uczniów z wielu przedmiotów w jednej czytelnej tabeli.

Struktura zapytania krzyżowego:

Zapytanie krzyżowe w Access SQL ma specyficzną budowę, która zawsze składa się z trzech części:

  1. TRANSFORM: Określa, co ma być obliczone (np. SUM, AVG, COUNT). To są dane "na środku" tabeli.
  2. SELECT: Określa, co ma być wierszami (np. Nazwisko ucznia).
  3. PIVOT: Określa, co ma stać się nagłówkami kolumn (np. Nazwy przedmiotów).

Przykład: Arkusz ocen (Macierz)

Załóżmy, że mamy tabelę Wyniki z kolumnami: Uczeń, Przedmiot, Ocena.

TRANSFORM FIRST(Ocena)
SELECT Uczeń
FROM Wyniki
GROUP BY Uczeń
PIVOT Przedmiot;
Dlaczego FIRST? W zapytaniu krzyżowym musisz użyć funkcji agregującej. Jeśli uczeń ma tylko jedną ocenę z przedmiotu, funkcja FIRST (pierwsza znaleziona) po prostu ją wyświetli. Jeśli chcesz średnią, użyjesz AVG.
---

3. Kiedy używać którego narzędzia?

Narzędzie Zastosowanie Zaleta
Parametry Szybkie filtrowanie "na żądanie" Użytkownik nie musi znać SQL
Crosstab Analiza porównawcza, statystyki Bardzo czytelny wynik (kompaktowy)
Zadanie egzaminacyjne:

Stwórz zapytanie krzyżowe, które wyświetli sumę sprzedaży (pole Kwota) dla każdego Sprzedawcy (wiersze), rozbitą na poszczególne Miesiące (kolumny).

Pokaż rozwiązanie
TRANSFORM SUM(Kwota)
SELECT Sprzedawca
FROM Sprzedaż
GROUP BY Sprzedawca
PIVOT Miesiąc;
